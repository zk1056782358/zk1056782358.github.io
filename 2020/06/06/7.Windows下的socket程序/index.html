bind函数原型

参数说明

  • s [in]

    标识未绑定套接字的描述符。

  • name [in]

    指向要分配给绑定套接字的本地地址的sockaddr结构的指针。

  • namelen [in]

    名称参数指向的值的长度（以字节为单位）

返回值

​ 如果没有错误发生，则绑定返回零。 否则，它返回SOCKET_ERROR

摘要

通过调用套接字函数创建套接字时，该套接字存在于名称空间（地址族）中，但未分配名称。 通过将本地名称分配给未命名的套接字，可以使用bind函数建立套接字的本地关联。使用Internet地址族时，名称由三部分组成：

  • 地址族。
  • 主机地址。
  • 标识应用程序的端口号。

如果应用程序不关心分配了什么本地地址，请在name参数的sa_data成员中为IPv4本地地址指定常量值INADDR_ANY，对于TCP / IP，如果将端口指定为零，则服务提供商会从动态客户端端口范围为应用程序分配一个唯一的端口。

调用bind之后，应用程序可以使用getsockname来获取分配给套接字的地址和端口。 如果Internet地址等于INADDR_ANY或in6addr_any，则在连接套接字之前getsockname不一定会提供地址，因为如果主机是多宿主的，则几个地址都可以有效。 对于客户端应用程序，建议不要绑定到端口0以外的特定端口号，因为存在与已经在本地计算机上使用该端口号的另一个套接字冲突的危险。

对于多播操作，首选方法是调用bind函数，将套接字与本地IP地址关联，然后加入多播组。尽管此操作顺序不是强制性的，但强烈建议您这样做。因此，多播应用程序将首先在本地计算机上选择一个IPv4或IPv6地址，通配符IPv4地址（INADDR_ANY）或通配符IPv6地址（in6addr_any）。然后，多播应用程序将使用name参数的sa_data成员中的地址中的该地址调用bind函数，以将本地IP地址与套接字相关联。如果指定了通配符地址，则Windows将选择要使用的本地IP地址。绑定功能完成后，应用程序将加入感兴趣的多播组。

inet_addr函数原型

参数说明

  • cp [in]

    以NULL终止的字符串，表示以Internet标准“。’’（点）表示法表示的数字。

返回值

如果没有错误发生，inet_addr函数将返回一个无符号的long值，其中包含给定Internet地址的适当二进制表示形式。如果cp参数中的字符串不包含合法的Internet地址，例如，如果“ a.b.c.d”地址的一部分超过255，则inet_addr返回值INADDR_NONE。

inet_addr函数解释cp参数指定的字符串。 此字符串表示以Internet标准“.”表示的数字Internet地址。返回的值是适合用作Internet地址的数字。所有Internet地址均以IP的网络顺序（从左到右按字节顺序）返回。 如果将“”（空格）传递给inet_addr函数，则inet_addr返回零。

注意点分数字可以是C语言中指定的十进制，八进制或十六进制。 以“ 0x”或“ 0X”开头的数字表示十六进制。 以“ 0”开头的数字表示八进制。 所有其他数字都解释为十进制。

htons函数原型

参数说明

  • hostshort [in]

    以主机字节顺序的16-bit数字。

返回值

htons函数以TCP / IP网络字节顺序返回该值。htons函数以主机字节顺序获取16位数字，并以TCP / IP网络（AF_INET或AF_INET6地址族）中使用的网络字节顺序返回16位数字。

htons函数可用于将主机字节顺序的IP端口号转换为网络字节顺序的IP端口号。

listen函数原型

参数说明

  • s [in]

    标识绑定的未连接套接字的描述符

  • backlog [in]

    挂起的连接队列的最大长度。 如果设置为SOMAXCONN，则负责套接字s的基础服务提供商将积压设置为最大合理值。 没有标准条款来获取实际积压值。

返回值

如果没有错误发生，则listen返回零。 否则，返回值SOCKET_ERROR

摘要

为了接受连接，首先使用套接字函数创建一个套接字，然后使用bind函数将其绑定到本地地址。用listen指定传入连接的待办事项，然后使用accept函数接受连接。面向连接的套接字（例如SOCK_STREAM类型的套接字）与listen一起使用。套接字s进入被动模式，在该模式下，传入的连接请求被确认并排队等待进程接受。SOMAXCONN的积压值是一个特殊的常数，它指示负责套接字的基础服务提供者将挂起的连接队列的长度设置为最大合理值。在Windows套接字2上，此最大值默认为一个较大的值（通常为数百或更大）。在Bluetooth应用程序中调用listen函数时，强烈建议为积压参数使用一个低得多的值（通常为2到4），因为仅接受少数客户端连接。这减少了分配给侦听套接字使用的系统资源。该建议同样适用于仅需要几个客户端连接的其他网络应用程序。侦听功能通常由一次可以具有多个连接请求的服务器使用。如果连接请求到达并且队列已满，则客户端将收到带有WSAECONNREFUSED指示的错误。

accept函数原型

参数说明

  • s [in]

    用于标识已使用侦听功能置于侦听状态的套接字。 实际上，连接是通过accept返回的套接字建立的。

  • addr [out]

    可选的指向缓冲区的指针，该缓冲区接收连接实体的地址。addr参数的确切格式由创建sockaddr结构的套接字时建立的地址族确定。可以不指定直接填NULL。

  • addrlen [in, out]

    可选的指向整数的指针，该整数包含addr参数指向的结构的长度。

返回值

如果没有发生错误，accept将返回SOCKET类型的值，该值是新套接字的描述符。 该返回值是实际建立连接的套接字的句柄。否则，将返回INVALID_SOCKET的值，并且可以通过调用WSAGetLastError来检索特定的错误代码。addrlen引用的整数最初包含addr指向的空间大小。 返回时，它将包含返回地址的实际长度（以字节为单位）

摘要

如果队列上没有挂起的连接，套接字就会被标记为阻塞(blocking)，那么accept函数可以阻塞调用者，直到出现连接为止。如果套接字被标记为非阻塞，并且队列上没有挂起的连接，则accept将返回错误，成功完成accept返回新的套接字句柄后，不能将接受的套接字用于接受更多连接。原始套接字保持打开状态，并侦听新的连接请求。参数addr是输出参数，该参数填充有通信层已知的连接实体的地址。 addr参数的确切格式由发生通信的地址族确定。addrlen是一个值输出参数；它最初应包含addr指向的大小；返回时，它将包含返回地址的实际长度（以字节为单位）。accept函数与面向连接的套接字类型（例如SOCK_STREAM）一起使用。如果addr和 / 或addrlen等于NULL，则不返回有关已接受套接字的远程地址的信息。

recv函数原型

参数说明

  • s[in]

    标识一个连接的套接字

  • buf [out]

    指向接受数据的缓冲区

  • len [in]

    缓冲区的长度

  • flags [in]

    标志位

返回值

三种返回值。
如果未发生错误，则recv返回接收到的字节数，并且buf参数指向的缓冲区将包含接收到的该数据。
如果已正常关闭连接，则返回值为零。
否则，返回值SOCKET_ERROR， 通过WSAGetLastError()函数获取具体错误代码

摘要

recv函数用于读取面向连接的套接字或无连接套接字上的传入数据。使用面向连接的协议时，必须在调用recv之前连接套接字。使用无连接协议时，必须在调用recv之前绑定套接字。对于已连接或未连接的套接字，recv函数会限制接收接收消息的地址。该功能仅从连接中指定的远程地址返回消息。来自其他地址的消息被(静默)丢弃。对于面向连接的套接字（例如，类型为SOCK_STREAM），调用recv将返回当前可用的尽可能多的数据-达到指定缓冲区的大小。如果已将套接字配置为在线接收OOB数据（套接字选项SO_OOBINLINE），但尚未读取OOB数据，则仅返回OOB数据。应用程序可以使用ioctlsocket或WSAIoctl SIOCATMARK命令来确定是否还有更多OOB数据需要读取。对于无连接套接字（SOCK_DGRAM类型或其他面向消息的套接字），将从连接功能指定的目标地址的第一个排队数据报(消息)中提取数据。如果数据报或消息大于指定的缓冲区，则缓冲区将填充数据报的第一部分，并且recv会生成错误WSAEMSGSIZE。对于不可靠的协议(例如UDP)，多余的数据将丢失；对于可靠的协议，服务提供商将保留数据，直到通过调用具有足够大缓冲区的recv成功读取数据为止。

如果套接字上没有可用的传入数据，则recv调用将阻塞，并根据为WSARecv定义的阻塞规则等待数据到达，除非套接字未阻塞，否则未设置MSG_PARTIAL标志。在这种情况下，将返回错误代码设置为WSAEWOULDBLOCK的SOCKET_ERROR值。 select，WSAAsyncSelect或WSAEventSelect函数可用于确定何时有更多数据到达。

flags参数可用于影响函数调用的行为 此函数的语义由套接字选项和flags参数确定。flags参数的可能值是通过将按位或运算符与以下任何值一起使用来构造的。

MSG_PEEK
窥视传入的数据。 数据被复制到缓冲区中，但不会从输入队列中删除。
该函数随后返回对recv（或recvfrom）函数的单次调用中可读取的数据量，该数据量可能与套接字上排队的数据总量不同。/在对recv（或recvfrom）函数的单个调用中实际可以读取的数据量被限制为在send或sendto函数调用中写入的数据大小。

send函数原型

参数说明

  • s [in]

    标识一个连接的套接字

  • buf [in]

    指向要被传输的缓冲区指针

  • len [in]

    缓冲区长度

  • flag[int]

    标志位

返回值

如果未发生错误，则send返回已发送的字节总数，该总数可以小于len参数中请求发送的字节数。否则，将返回SOCKET_ERROR的值，并且可以通过调用WSAGetLastError来检索特定的错误代码。

摘要

对于面向消息的套接字（例如，AF_INET或AF_INET6的地址系列，SOCK_DGRAM的类型以及IPPROTO_UDP的协议），必须注意不要超过基础提供程序的最大数据包大小。提供者的最大消息包大小可以通过调用optname参数设置为SO_MAX_MSG_SIZE的getsockopt来检索套接字选项的值来获得。如果数据太长而无法自动通过基础协议，则返回错误WSAEMSGSIZE，并且不会传输任何数据。成功完成发送功能并不表示数据已成功传递并接收到接收者。此功能仅指示数据已成功发送。

如果传输系统中没有可用的缓冲区空间来保存要传输的数据，则除非套接字已置于非阻塞模式，否则send将阻塞。在面向非阻塞流的套接字上，写入的字节数可以在1到请求的长度之间，具体取决于客户端和服务器计算机上的缓冲区可用性。select，WSAAsyncSelect或WSAEventSelect函数可用于确定何时可以发送更多数据，允许使用len参数为零的调用send，实现将其视为成功。在这种情况下，发送将返回零作为有效值。对于面向消息的套接字，将发送零长度的传输数据报。

shutdown函数原型

参数说明

  • s[int]

    标识一个socket

  • flag[int]

    标志位

返回值

If no error occurs, shutdown returns zero. Otherwise, a value of SOCKET_ERROR is returned

摘要

关闭功能可用于所有类型的套接字，以禁用接收和 / 或传输。如果how参数是SD_RECEIVE，则不允许随后调用套接字上的recv函数。这对较低的协议层没有影响。对于TCP套接字，如果套接字上仍有等待接收的数据排队，或者数据随后到达，则将重置连接，因为无法将数据传递给用户。对于UDP套接字，传入的数据报被接受并排队。在任何情况下都不会生成ICMP错误数据包。如果how参数为SD_SEND，则不允许后续调用send函数。对于TCP套接字，将在接收器发送并确认所有数据之后发送FIN。设置SD_BOTH的方式将同时禁用发送和接收，如上所述。

关闭功能不会关闭套接字。在调用closesocket之前，不会释放连接到套接字的任何资源。为了确保所有数据在关闭之前在连接的套接字上发送和接收，应用程序应在调用closesocket之前使用shutdown关闭连接。

等待通知远端已发送所有数据并启动正常断开连接的一种方法是使用WSAEventSelect函数，

如下所示：

  • 调用WSAEventSelect来注册FD_CLOSE通知。
  • 使用how = SD_SEND调用关机。接收到FD_CLOSE时
  • 调用recv或WSARecv，直到函数成功完成并指示已接收到零字节。如果返回了SOCKET_ERROR，则无法正常断开连接。
  • 调用closesocket。

等待通知远端已发送所有数据并发起正常断开连接的另一种方法是使用重叠的接收调用：

  • 使用how = SD_SEND调用关机。

  • 调用recv或WSARecv，直到函数成功完成并指示已接收到零字节。如果返回了SOCKET_ERROR，则无法正常断开连接。

  • 调用closesocket。

注意无论套接字上的SO_LINGER设置如何，关闭功能都不会阻塞。一旦调用了关闭功能以禁用发送，接收或两者都禁用，就没有方法重新为现有套接字连接启用发送或接收。

在关闭套接字后，应用程序不应依赖于能够重用它。特别是，不需要Windows Sockets提供程序来支持在已关闭的套接字上使用connect。

如果应用程序要重用套接字，则应在dwFlags参数设置为TF_REUSE_SOCKET的情况下调用DisconnectEx函数，以关闭套接字上的连接并准备要重用的套接字句柄。当DisconnectEx请求完成时，可以将套接字句柄传递给AcceptEx或ConnectEx函数。

如果应用程序要重用套接字，则可以在将所有数据都排队等待传输并准备要重用的套接字句柄之后，通过将dwFlags参数设置为TF_DISCONNECT和TF_REUSE_SOCKET来调用TransmitFile或TransmitPackets函数。当TransmitFile请求完成时，可以将套接字句柄传递给先前用于建立连接的函数调用，例如AcceptEx或ConnectEx。 TransmitPackets函数完成后，可以将套接字句柄传递给AcceptEx函数。

closesocket函数原型

参数说明

  • s [in]

    标识一个套接字

返回值

If no error occurs, closesocket returns zero. Otherwise, a value of SOCKET_ERROR is returned

摘要

closesocket函数关闭套接字。用它释放在s参数中传递的套接字描述符。请注意，一旦发出closesocket函数，在s参数中传递的套接字描述符可能会立即被系统重用。因此，期望对s参数中传递的套接字描述符的进一步引用失败并出现错误WSAENOTSOCK是不可靠的。Winsock客户端绝不能与另一个Winsock函数调用同时发出closesocket。
